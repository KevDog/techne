# Project Rules

- No code before requirements are documented
- No architecture before requirements are reviewed
- Every critical decision gets an ADR entry in `DECISIONS.md`
- Theatrical domain terms must be defined in `GLOSSARY.md` before use in code
- Never work sequentially on independent tasks — always check for parallelization
- All front-end work must use the `frontend-design` agent
- Claude AI features are optional per org — app must be fully functional without them
- Roles are never hardcoded — flexibility is a first-class requirement
